# Review Questions: SDLC

**1. What is SDLC?**
SDLC (Software Development Life Cycle) is a structured, step-by-step process used by software engineering teams to design, develop, test, and deploy high-quality software. It provides a standardized framework that guides the entire software development journey from initial conception to final retirement.

**2. Why is SDLC important?**
SDLC is important because it provides a clear, structured methodology for building software, which ensures the final product meets customer expectations, is delivered on time, and stays within budget. It helps teams manage complex projects, control costs, minimize risks, and maintain a high standard of quality through organized phases and reviews.

**3. List the seven phases of SDLC.**
1. Planning
2. Requirement Analysis
3. Design
4. Development (Coding)
5. Testing
6. Deployment
7. Maintenance

**4. Differentiate between functional and non-functional requirements.**
- **Functional Requirements:** Define *what* the system should do. They specify the features, behaviors, and business rules of the software (e.g., "The system must allow a user to reset their password").
- **Non-functional Requirements:** Define *how* the system should perform. They specify the quality attributes, constraints, and performance metrics (e.g., "The password reset email must be sent within 5 seconds," or requirements regarding scalability, security, and usability).

**5. What happens during the testing phase?**
During the testing phase, the newly developed software is rigorously evaluated to identify and fix defects (bugs). Quality Assurance (QA) professionals and testers verify that the application meets all the defined requirements, functions correctly, handles edge cases, and is secure before it reaches the end-users.

**6. Explain the difference between deployment and maintenance.**
- **Deployment:** The one-time (or periodic) event of releasing the finalized, tested software into the production environment so that end-users can access and use it.
- **Maintenance:** The ongoing, continuous phase *after* deployment. It involves monitoring the software, fixing newly discovered bugs, releasing security patches, and updating the software to add new features or adapt to new operating systems.

**7. Compare the Waterfall and Agile models.**
- **Waterfall Model:** A linear, sequential approach where each phase (Planning, Design, Coding, etc.) must be fully completed before the next one begins. It is rigid, requires extensive upfront documentation, and is best for projects with strict, unchanging requirements.
- **Agile Model:** An iterative and incremental approach that emphasizes flexibility and continuous feedback. Development is broken down into small, manageable cycles (sprints), allowing teams to adapt to changing requirements and deliver functional software rapidly.

**8. Who are the major stakeholders involved in SDLC?**
Major stakeholders typically include:
- Clients / Business Owners (who sponsor the project)
- End-users (who will use the software)
- Project Managers (who oversee the timeline and resources)
- Business Analysts (who gather requirements)
- Software Developers / Engineers (who build the software)
- UI/UX Designers (who design the interface)
- QA Testers (who ensure quality)

**9. What is User Acceptance Testing (UAT)?**
User Acceptance Testing (UAT) is the final testing phase before deployment, where the actual software users (or clients) test the application in a real-world scenario. The primary goal of UAT is to validate that the software meets the business needs and is ready for everyday use.

**10. Why is maintenance important after deployment?**
Maintenance is crucial because software environments and user needs are constantly evolving. It ensures the software remains functional by patching security vulnerabilities, fixing latent bugs that users discover, ensuring compatibility with new hardware or OS updates, and adding new features to keep the software relevant and valuable.
